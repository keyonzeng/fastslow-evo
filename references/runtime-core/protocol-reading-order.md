# FastSlow Protocol Reading Order

Use this reading order in OpenClaw when working on FastSlow evidence or candidates.

## If the task is about new evidence
1. `references/runtime-optional/openclaw-judgment-first.md`
2. `references/runtime-branches/evidence-protocol.md`
3. `references/runtime-core/review-protocol.md`
4. `references/runtime-core/evaluation-rubric.md`
5. relevant evidence markdown files
6. existing candidate markdown files if sameness is in question
7. `references/runtime-core/review-output-format.md`

## If the task is about candidate review
1. `references/runtime-optional/openclaw-judgment-first.md`
2. `references/runtime-branches/candidate-protocol.md`
3. `references/runtime-core/review-protocol.md`
4. `references/runtime-core/evaluation-rubric.md`
5. `references/runtime-branches/candidate-review-checklist.md`
6. linked evidence markdown files
7. target candidate markdown files
8. `references/runtime-core/review-output-format.md`

## If the task is about heartbeat review
1. `references/runtime-optional/openclaw-judgment-first.md`
2. `references/runtime-branches/heartbeat-protocol.md`
3. `references/runtime-core/review-protocol.md`
4. `references/runtime-core/evaluation-rubric.md`
5. recent evidence markdown files
6. current candidate markdown files
7. `references/runtime-core/review-output-format.md`

## Default Rule
If you are about to judge pattern sameness, convergence, or promotion without first grounding in protocol, stop and read the relevant protocol files.
