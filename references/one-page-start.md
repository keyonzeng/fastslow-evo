# One-Page Start

Use this page when you want the shortest path to value.

## What FastSlow Evo does

FastSlow Evo is a dual-loop self-evolution system.

- **Fast loop**: fix local repeated mistakes quickly
- **Slow loop**: promote proven fixes into durable capability

## Start in 5 steps

### 1. Capture one real issue
Use an incident, correction, or repeated friction.

### 2. Route it
Ask:
- Is it local, frequent, low-risk, and easy to verify next time?
  - Use **Quick Adapt**
- Is it recurring across tasks, broader in scope, or higher risk?
  - Use **Slow Promote**

### 3. Apply the smallest fix
Preferred fast outputs:
- checklist
- template
- memory rule
- tiny validation rule
- tiny script

### 4. Reuse on the next similar task
If it helps immediately, keep it.

### 5. Promote only after proof
Promote when:
- it recurs 3+ times
- it works across contexts
- it passes validation
- it does not introduce regressions

## Default Rule

Do not make a new skill or broad policy from one event.

## Best first move

If unsure, start with **Quick Adapt**, not Slow Promote.
