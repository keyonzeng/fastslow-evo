# One-Page Start

Use this page when you want the shortest path to value.

## What FastSlow Evo does

FastSlow Evo is a dual-loop self-evolution system.

- **Fast loop**: fix local repeated mistakes quickly
- **Slow loop**: promote proven fixes into durable capability

## Start in 5 steps

### 1. Notice one real signal
Use a correction, incident, repeated friction, or a clearly reusable win.

### 2. Preserve evidence with context
Write the evidence so the host model can review it later.
Use `references/runtime-branches/evidence-protocol.md`.
Do not force pattern judgment too early.

### 3. Let the model review and route it
Use `references/runtime-core/review-protocol.md` and `references/runtime-core/evaluation-rubric.md`.
Ask:
- Is it local, frequent, low-risk, and easy to verify next time?
  - Use **Quick Adapt**
- Is it recurring across tasks, broader in scope, or higher risk?
  - Use **Slow Candidate Review**
- Is evidence still weak?
  - Use **Observe**
- Is it noise or a bad candidate?
  - Ignore or Reject it

### 4. Apply the smallest justified next step
Preferred fast outputs:
- checklist
- template
- memory rule
- tiny validation rule
- small procedural tweak

### 5. Promote only after proof
Promote when:
- it recurs 3+ times
- it is clearly the same underlying pattern
- it works across contexts
- it passes validation
- it does not introduce regressions

## Default Rule

Do not make a new skill or broad policy from one event.

## Best first move

If unsure, start with **Quick Adapt**, not Slow Promote.
