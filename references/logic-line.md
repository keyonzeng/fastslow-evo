# FastSlow Evo Logic Line

If this project starts to feel complicated, come back to this page.

## One-sentence definition

FastSlow Evo is a dual-loop self-evolution system that drives agent growth through fast local adaptation and slow durable promotion.

## The whole logic in 6 lines

1. Something happens.
2. Decide whether it matters.
3. If it matters, decide whether it should go fast, slow, stay observed, or be ignored.
4. If it should be kept, preserve the minimum useful evidence.
5. If it keeps working or keeps recurring, review it again.
6. Only then promote it into durable capability.

## The four buckets

### Fast
Use when a small local fix will probably help next time.

### Slow
Use when the pattern is stable enough to deserve durable promotion.

### Observe
Use when the signal may matter, but evidence is still thin.

### Ignore
Use when the event is noise and acting would add more complexity than value.

## What FastSlow Evo is not

- not a logging bucket
- not a memory dump
- not a self-modification engine
- not a Python rule engine with a skill wrapper

## What it really is

A judgment layer.

It decides whether experience should stay local, become durable, wait for more evidence, or be discarded.

## The two loops

### Fast loop
Small local improvement now.

### Slow loop
Durable promotion later, but only after proof.

## Why this is useful

Without the fast loop, useful corrections evaporate.
Without the slow loop, useful patterns never become capability.
Without observe/ignore, the system becomes noisy and overfit.

That is the whole idea.
