# Evolution Protocol

Use this protocol to turn evidence into durable improvement.

## Stage 0 — Evidence Threshold

Do not promote from speculation.

Minimum thresholds:
- one strong incident can justify a note or local warning
- two similar incidents can justify a checklist or validation spec
- three recurring incidents can justify a script, workflow rule, or skill proposal

## Stage 1 — Gap Typing

Assign one primary gap type and optional secondary tags.

## Stage 2 — Spec Delta Drafting

Draft the smallest change that would have prevented or reduced recurrence.

## Stage 3 — Validation Design

Define how the delta will be checked before wider adoption.

## Stage 4 — Review Gate

Review levels:
- L0: evidence capture only
- L1: curated notes, checklists, templates
- L2: scripts, skill edits, durable workflow rules
- L3: code/config mutation, automation triggers, anything with meaningful blast radius

## Stage 5 — Materialization

Materialize to the lightest sufficient artifact.

## Stage 6 — Regression Guard

Check for:
- higher complexity
- broader scope than intended
- contradiction with existing specs
- new safety risks
- user-friction increase

## Stage 7 — Measurement

Compare before vs after using a scorecard.

## Hard Prohibitions

Do not automatically:
- rewrite the evolution system itself in production
- touch secrets or unrelated histories
- publish packages or skills
- run infinite evolution loops on live systems
