# Fast / Slow Evolution Model

`fastslow-evo` is built around two linked forms of improvement.

## Fast Loop

Purpose:
Absorb local experience quickly.

Best for:
- repeated small mistakes
- user corrections
- missing checklist items
- tone or format drift
- omitted action items
- minor workflow friction

Typical outputs:
- checklist
- template
- memory rule
- tiny validation rule
- tiny script

Success test:
Did the agent make the next similar task easier, safer, or less error-prone within the same day or next few tasks?

## Slow Loop

Purpose:
Promote validated local improvements into durable higher-order capability.

Best for:
- cross-task recurring patterns
- capability boundary upgrades
- reusable workflow rules
- behavior policies
- skill-level abstractions

Typical outputs:
- capability spec
- behavior spec
- validation spec
- evolution rule
- reusable skill or workflow

Success test:
Did a proven local fix become a stable, reusable improvement without causing regressions?

## Key Principle

Fast loop handles **quantity of adaptation**.
Slow loop handles **quality of promotion**.

Or more simply:
- fast loop = adapt
- slow loop = promote

## Design Rule

Do not force everything into the slow loop.
Do not let everything stay in the fast loop.

Healthy evolution depends on movement between the two.
