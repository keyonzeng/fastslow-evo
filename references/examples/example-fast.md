# Example: Fast route for a local correction

## Situation

The user corrects the agent's reply style:
- the answer was too long
- the core content was acceptable
- no privacy, workflow-boundary, or capability-boundary issue is involved

This is the second similar correction within a short span.

## What happened

The pattern is not a broad capability failure.
It is a local delivery mismatch with near-term recurrence risk.

## Why it matters

If left alone, the same friction is likely to repeat in the next few tasks.
A small adjustment is enough.
A durable promotion would be premature.

## Sameness judgment

Treat this as the same underlying pattern if prior evidence shows the same delivery mismatch:
- overlong answers
- too much framing before the conclusion
- weak compression despite the same user preference

Do not split it into multiple candidates just because wording differs.

## Route

`fast`

## Smallest justified action

- add or update a tiny local rule for concise-first delivery
- apply it in the next task
- monitor whether recurrence drops

## Why not slow candidate

- the issue is still local
- the fix is easy to test immediately
- durable promotion would add more process than value

## Example write-back

- outcome: `fast`
- rationale: repeated local delivery mismatch; low risk; immediate validation available
- action: tighten concise-first output in the next tasks; monitor recurrence
