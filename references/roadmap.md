# Roadmap

## Goal

Turn FastSlow Evo into a durable OpenClaw skill with a portable method and a clear first-wave user fit.

## Phase 1 — Core clarity

Complete:
- naming
- dual-loop model
- starter guides
- examples
- fast/slow routing baseline

## Phase 2 — OpenClaw-native hardening

Next priorities:
- memory evolution rules for workspace + MEMORY.md + daily notes
- messaging behavior evolution across group/private/direct contexts
- delivery evolution rules for current chat vs docs vs files
- tooling evolution rules for result faithfulness and execution proof

## Phase 3 — Builder adoption

Add:
- stronger recurrence detection
- better promotion heuristics
- more reusable workflow examples
- coding/workflow-specific quick-adapt packs

## Phase 4 — Mainstream packs

Add lightweight fronts for:
- meetings
- email drafting
- reading notes
- study recall

These should consume the system without exposing all of its internal governance.

## Non-Goals for now

- full autonomous self-modification
- broad general-consumer simplification of the core system
- infinite-loop optimization in production
- treating every repeated annoyance as a skill candidate
