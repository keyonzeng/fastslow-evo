# Quick Adapt

Use the fast loop when the goal is immediate improvement on the next similar task.

## When to use

Use Quick Adapt when the issue is:
- local
- frequent or likely to recur soon
- low blast radius
- easy to verify in the next task
- solvable without changing global capability boundaries

Examples:
- summary missed an action item
- tone was too formal or too generic
- a tool result was misread
- follow-up format was wrong
- user said “next time do it this way”

## Inputs

- one incident, correction, or successful workaround
- one primary gap type
- one concrete next-task success condition

## Output

A **tiny fix**, chosen from:
- checklist item
- template tweak
- memory rule
- tiny validation rule
- tiny script

## Workflow

1. Capture the event.
2. Name the primary gap.
3. Ask: what is the smallest durable fix?
4. Apply only that fix.
5. Reuse it in the next similar task.
6. Check whether recurrence drops.

## Rule

Do not create a new skill, global rule, or broad capability claim from one event.

## Success Criterion

The next similar task becomes easier, safer, faster, or less error-prone within the same day or next few tasks.
